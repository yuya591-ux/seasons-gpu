# 外部AI生成の組み込み方針（両プロジェクト共通の土台）

> 目的: 自分のPCの性能（RTX 4050 / VRAM 6GB）に縛られず、画像・テクスチャを外部の無料AIに生成させて、クオリティの壁を越える。
> 大原則: 完全無料で・安全に・Claude Codeに丸投げで・後から壊れない形で。

---

## 1. なぜこの方式なら6GBのPCでも完全無料で動くのか
- 画像生成は「自分のPCのGPU」ではなく「外部サービスのクラウド」で計算される → 自分のPCの性能は無関係
- 使うのは **Pollinations の Flux モデル**（一次情報で「完全無料・常時無料」を確認済み・2026年6月時点）
- ただし料金体系は変動中（6/22に値下げ予定等）→ 組み込み前に最新を確認する手順を必須にする

## 2. 絶対に守る課金ゼロのルール（Claude Codeに厳守させる）
- **使うモデルは Flux のみに固定**（無料）。GPT Image / Gemini / Seedream 等の有料モデルは使わない
- 有料の Pollen を消費する操作を一切させない。クレジットカード登録もしない
- 組み込み前に、Claude Code に必ず最新の料金・利用規約を確認させる:
  - 「https://enter.pollinations.ai/ と Pollinations の最新ドキュメントを確認し、Fluxモデルの画像生成が現在も無料か、商用・趣味利用の制約、レート制限を調べて報告してから組み込んで。少しでも課金が発生し得る設計なら、その箇所を先に教えて」

## 3. 安全性とdurability（自分がいなくなっても動く原則）
- **生成は「開発時に一度だけ」。本番のアプリ/ゲームにAPIや鍵を埋め込まない**
  - 開発時: Claude Code が Pollinations を呼んで画像/テクスチャを生成 → ファイルとして保存（assets/ に置く）
  - 本番: 保存済みの画像ファイルを表示するだけ。実行中に外部APIを叩かない
- この方式の利点:
  - APIキーがアプリに含まれない（漏洩・不正利用なし）
  - Pollinationsが将来有料化・終了しても、保存済み画像でアプリは動き続ける
  - 個人アカウント・個人キーに依存しない
- 鍵が必要な場合でも、生成スクリプト側（開発環境）にのみ置き、`.gitignore` で除外してGitHubに上げない

## 4. ライセンス・著作権
- Flux生成画像の商用利用可否・権利を、上記の規約確認時に合わせてチェックさせる（趣味利用なら問題ないが、念のため記録）
- 生成物の出典（Pollinations / Flux / 生成日 / プロンプト）を `CREDITS.md` に記録
- 既存作品（僕の夏休み等）の固有の絵・キャラを模倣するプロンプトは使わない。あくまでオリジナルの雰囲気

## 5. Claude Code への丸投げ範囲（変わらず）
- 生成スクリプトの作成、Pollinations呼び出し、画像保存、アプリへの組み込み、デプロイまで全部Claude Code
- 人間の手作業は、もし鍵取得でブラウザ操作が要る場合のその1回だけ（Fluxの基本利用は鍵不要の可能性が高い。要確認）
