# CLAUDE.md — 「眺めて整う、季節と天気のアンビエント」恒久ルール

## プロジェクト概要
- 個人の趣味プロジェクト。販売・公開予定なし。ただし「宣伝されて商品化できそうな見た目」を本気で狙う
- 四季 × 天気 × 時間帯 × 環境音 を選び、ただ眺めて心を整える癒しアプリ
- 目的は「眺める・聴く」だけで完成する没入感と、現象としての本物の質感。ゲーム的なクリア条件は置かない
- 詳細仕様は `SPEC.md` を正とする。本ファイルは全期間で守る恒久ルール

## 最重要の作り方（詰め込みすぎを避ける規律）
- **器（構造）は最初から全部入りで作る**: 四季・天気・時間帯・音を「データで持つ器」として設計し、選択メニューも全部並べる
- **中身（シーン）は1枚ずつ、商品レベルに化けるまで作り込んで増やす**: 最初に1枚だけを完璧に仕上げ、それを見本に他を量産する
- 最初の見本シーン = **夏・雨・夕方・ヒグラシ**
- 新しいシーンの追加が「色データ＋音ファイルの追加だけ」で済む構造を死守する（既存を壊さず足す＝非破壊・疎結合）

## 技術スタック（固定）
- 描画は **WebGL（シェーダー／GLSL）** を主役にする。雨粒の屈折・水面・光・霧・粒子などの「現象」を、画像を使わず計算で生成する
  - ライブラリは Three.js もしくは軽量なフルスクリーン・シェーダー構成。Claude Codeが最も確実に高品質を出せる方を選び、理由をREADMEに記載
  - 重い動画ファイルや生成画像に依存しない。質感はすべてシェーダー計算で出す
- UI部分は HTML + CSS + JavaScript。ビルドは Vite
- 音声は Web Audio API / HTML Audio。環境音はフリー素材（CC0等）をループ再生
- ホスティングは **GitHub Pages**（自動デプロイ。追加認証不要）
- **外部API非依存・課金なし・個人キー非依存**（「自分がいなくなっても動く」原則）

## 性能（シェーダー特有の注意）
- スマホでも滑らかに動くこと（フレーム落ち・発熱に注意）。重い計算は品質設定で落とせるようにする
- WebGL未対応・低性能端末向けに、簡易表示へのフォールバックを用意する
- バッテリー配慮: 非アクティブ時は描画を間引く／止める

## 開発プロセス
1. **Phase 0 必須**: 実装前に、データ構造案（季節×天気×時間帯×音の器）・画面ワイヤーフレーム・シェーダーの描画方針（雨粒の屈折をどう計算するか等）・最初の見本シーンの色設計を提示し、人間承認後に着手
2. 各フェーズ着手前に改善案を1〜3個提示する
3. **investigate-first**: 既存コード変更前に構造・関数名を調査
4. 実装後の**自己検証必須**: ビルド／lint／headlessブラウザでの表示確認／コンソールエラーなし／WebGLのフレームレート確認まで済ませてから報告
5. 人間の手作業は認証・デプロイ承認類のみ。まとめてチェックリスト形式で提示
6. デプロイまで Claude Code が実行

## 著作権・素材の誠実性（絶対遵守）
- 環境音（蝉・ヒグラシ・雨・雷・風鈴・波・虫の音・ウグイス等）は、ライセンスが明確なフリー素材（CC0最優先）のみ使用
- 出典・ライセンスURLを `CREDITS.md` に全数記録する。ライセンス不明の素材は使わない
- 既存アプリ・作品の固有の音/絵/名称を模倣しない
